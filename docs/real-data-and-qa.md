# 问答推荐 & 真实数据来源说明

## 当前已实现：问答式推荐

- **入口**：首页「问一问」输入框，例如输入「周四下班去附近爬抱石，价格低一点」后点「推荐」。
- **逻辑**：用关键词解析出「周几、时段、类型、价格偏好、区域」，再基于现有岩馆数据筛选并排序（如按单次价格从低到高），以一句话总结 + 列表形式展示。
- **数据来源**：仍是你后台维护的 mock / Supabase 数据（`gyms` 表 + 本地 `gyms-shanghai.json`），**没有**从外网实时抓取。

## 从网络抓取：分两阶段实现

### 阶段 1：聚合链接 / 标题 / 时间（已实现）

- **做法**：不抓正文，只存「某馆 RSS/页面 的链接、标题、发布时间」到 Supabase 表 `social_posts`。
- **已具备**：
  - 表结构见 `docs/latest-activity-setup.md`（与 `events`、`route_sets` 同页）。
  - 详情页「社交媒体动态（链接/标题/时间）」区块会读 `social_posts` 并展示。
  - 脚本 `scripts/fetch-rss-to-supabase.mjs`（`pnpm run fetch-rss`）可从 RSS 拉取并 upsert 到 `social_posts`，需配置环境变量。
- **文档**：`docs/fetcher-design.md`、`docs/latest-activity-setup.md`。

### 阶段 2：抓取结构化信息（开放时间、价格、活动）

- **目标**：自动写入开放时间、价格、活动到 `gyms` / `events`，供列表与问答使用。
- **难点**：各馆渠道不一（官网、公众号、美团等），无统一 API，反爬与结构变更需持续维护。
- **建议**：先完成阶段 1；阶段 2 只对 1～2 个稳定数据源做试点，写专用脚本写入 Supabase。
- **文档**：`docs/structured-scraping-phase2.md`。

### 若有第三方开放 API

- 若某平台提供场馆开放时间/价格的 API，可在服务端或定时脚本中请求后写入 `gyms`，问答与列表逻辑无需大改。

## 小结

- **问答**：已实现，基于当前 gym 数据；「周四下班、抱石、价格低」会得到按价格排序的推荐。
- **阶段 1（链接聚合）**：表 + 前端展示 + RSS 脚本已就绪，在 Supabase 建表并录入或跑脚本即可。
- **阶段 2（结构化抓取）**：见 `docs/structured-scraping-phase2.md`，建议在阶段 1 稳定后再选源试点。
